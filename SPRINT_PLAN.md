# Sprint Plan — ITG SmartHire

تاريخ البدء: ___/___/2026
المدة: 8 اسابيع (4 Sprints)

## الفريق والادوات

بشار — PMO + DevOps — VS Code, Docker Desktop, AWS CLI
عمر — Backend (Core) — Visual Studio 2022
مازن — Full-Stack (Dashboard + Admin) — Visual Studio 2022
غزل — Backend + Documentation — Visual Studio 2022

راما — UI/UX Design — Figma + Visual Studio 2022

## Sprint 0: التجهيز (اول 3 ايام)

بشار:
- انشاء GitHub repository واضافة الفريق
- رفع الملفات الاولية (README, Architecture, Git Guide, gitignore)
- انشاء project board على GitHub
- انشاء فرع develop
- تسجيل مفتاح Gemini API مجاني
- ارسال دليل Git للفريق
- التنسيق مع مشرف ITG

عمر:
- تثبيت الادوات (.NET 8 SDK, Visual Studio, Git)
- انشاء المشروع: dotnet new mvc -n SmartHire
- انشاء Solution ورفع الهيكل على GitHub

مازن:
- تثبيت الادوات (.NET 8 SDK, Visual Studio, Git)
- قراءة ملف Architecture وفهم صفحات Dashboard و Admin اللي رح يبنيها
- تجربة clone المشروع من GitHub

غزل:
- تثبيت الادوات (.NET 8 SDK, Visual Studio, Git)
- كتابة الموديلز الاساسية: User, Job, JobSkill, Application, AIAnalysis, SkillSynonym, SystemSetting
- التنسيق مع فرح على اسماء الحقول وانواعها

سدين:
- تثبيت الادوات (.NET 8 SDK, Visual Studio, Git)
- البحث عن كيفية استدعاء Gemini API من C# (HTTP POST request)
- تجربة ارسال طلب بسيط لـ Gemini API والحصول على رد

فرح:
- تثبيت SQL Server + SSMS + Git
- تصميم ERD باستخدام draw.io أو dbdiagram.io
- التنسيق مع غزل عشان الجداول تطابق الموديلز

راما:
- تثبيت Git
- رسم wireframes بـ Figma لكل الصفحات:
  تسجيل الدخول، قائمة الوظائف، نموذج التقديم،
  لوحة HR، تفاصيل المتقدم، مقارنة، احصائيات،
  انشاء وظيفة، اعدادات

## Sprint 1: الاساسيات (الاسبوع 1-2)

بشار:
- كتابة Dockerfile وتجربته
- متابعة الفريق على project board
- حل أي مشاكل Git عند الفريق
- اول تقرير للمشرف بنهاية الاسبوع 2

عمر:
- اعداد ASP.NET Identity (تسجيل + دخول + 3 صلاحيات)
- انشاء HomeController (الصفحة الرئيسية + قائمة الوظائف)
- انشاء AccountController (Login, Register, Logout)
- انشاء JobsController (عرض تفاصيل وظيفة)
- انشاء ApplicationsController (نموذج التقديم + حفظ + رفع CV)

مازن:
- انشاء DashboardController (هيكل اساسي)
- انشاء AdminController (هيكل اساسي)
- ربط الصفحات بالبيانات التجريبية من فرح

غزل:
- انشاء ViewModels لكل صفحة
- انشاء Service interfaces
- مساعدة عمر ومازن بالموديلز اذا احتاجوا تعديل
- بدء التوثيق

سدين:
- بناء CVParserService (استخراج نص من PDF باستخدام PdfPig)
- بناء الهيكل الاساسي لـ AIService (الاتصال بـ Gemini API)
- تجربة ارسال CV حقيقي لـ Gemini والحصول على JSON

فرح:
- انشاء AppDbContext مع كل الجداول
- انشاء اول Migration وتشغيلها
- اضافة بيانات تجريبية: 5 وظائف، 20 مهارة، 10 مرادفات، مستخدم admin و HR

راما:
- بناء _Layout.cshtml (Navbar + Footer + Bootstrap)
- بناء صفحة Login
- بناء صفحة Register
- بناء صفحة قائمة الوظائف (Index)
- بناء صفحة تفاصيل الوظيفة (Details)

---

## Sprint 2: التقديم + الفلترة (الاسبوع 3-4)

بشار:
- انشاء حساب AWS Free Tier
- تجربة رفع المشروع على EC2
- انشاء RDS و S3
- تقرير منتصف المشروع للمشرف

عمر:
- برمجة حفظ الطلب + رفع CV + تخزينه
- برمجة صفحة "طلباتي" للمتقدم
- برمجة ApplicationService مع الفلترة التلقائية

مازن:
- بناء لوحة تحكم HR كاملة (جدول + فلاتر + ازرار قبول/رفض)
- بناء صفحة تفاصيل المتقدم
- بناء صفحة انشاء وظيفة للادمن

غزل:
- برمجة الفلترة التلقائية (رفض الراتب الزائد والخبرة الناقصة)
- مساعدة مازن بربط Dashboard بالبيانات
- تحديث التوثيق

سدين:
- برمجة AIService كامل (ارسال CV لـ Gemini + استقبال JSON)
- برمجة MatchingService (حساب الدرجات الاربعة + الدرجة الكلية)
- ربط التحليل بعملية التقديم

فرح:
- اختبار حفظ الطلبات
- اختبار رفع الملفات
- اختبار الفلترة التلقائية بأمثلة مختلفة

راما:
- بناء صفحة نموذج التقديم (Apply) مع رفع ملف
- بناء صفحة طلباتي (MyApplications)
- تحسين لوحة HR مع مازن (الوان الدرجات، الترتيب)

---

## Sprint 3: الذكاء الاصطناعي + الميزات المبتكرة (الاسبوع 5-6)

بشار:
- اضافة Gemini API Key لاعدادات AWS
- اختبار AI على السيرفر
- مراقبة استهلاك API

عمر:
- ربط AIService بالتقديم (التحليل يشتغل تلقائي بعد التقديم)
- اصلاح أي مشاكل بالـ Controllers

مازن:
- بناء صفحة مقارنة المتقدمين (Compare)
- بناء صفحة الاحصائيات (Statistics) مع Chart.js
- بناء صفحة اعدادات النظام (Settings)

غزل:
- اضافة بيانات كافية لجدول SkillSynonyms
- مراجعة التوثيق والتأكد من تغطية كل الميزات
- مساعدة سدين بتنسيق JSON

سدين:
- برمجة AI Summary (توليد ملخص لكل متقدم)
- برمجة AI Interview Questions (توليد 5 أسئلة مخصصة)
- برمجة AI Job Description Generator (توليد وصف وظيفي)
- معالجة أخطاء Gemini API

فرح:
- اختبار دقة تحليل AI مع CVs مختلفة
- اختبار حساب الدرجات بنتائج معروفة مسبقا
- اختبار أسئلة المقابلة والملخصات

راما:
- بناء عرض أسئلة المقابلة بصفحة تفاصيل المتقدم
- بناء صفحة المقارنة (جدول بأعمدة متجاورة)
- بناء الرسوم البيانية بصفحة الاحصائيات
- اضافة AI Generator بصفحة انشاء الوظيفة

غزل:
- انهاء التوثيق الكامل

فرح:
- اختبار شامل لكل الصفحات والميزات
